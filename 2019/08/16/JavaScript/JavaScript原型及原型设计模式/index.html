JavaScript原型及原型设计模式

发表于 2019-08-16 更新于 2020-04-07

导读

JavaScript 常被描述为一种基于原型的语言；那什么是原型呢？原型是 JavaScript 中一个比较难理解的概念，原型相关的属性也比较多；当初自己学习时也是一头雾水，但却是进阶 JavaScript 必不可少的一步；像继承、设计模式等都涉及到原型的相关知识；希望你通过阅读本文，可以让你对原型及原型的设计有所认识。

原型

每一个 JavaScript 对象( Null 除外)在创建的时候就会与之关联另一个对象，而”另一个对象”指的就是原型；函数中使用的是 prototype 属性与原型关联，普通的对象则使用proto属性与原型关联。
在JavaScript中可使用 Object.getPrototypeOf(object) 来访问对象的原型。

原型对象

在 JavaScript 中，新创建一个函数，都会有一个 prototype 属性，这个属性指向函数的原型对象 prototype；普通对象没有 prototype 属性，但有proto属性。原型对象的用途是为每个实例对象存储共享的方法和属性，所以在原型上定义的属性和方法都会被其实例所共享。

person1 与 person2 都是构造函数 Person 创建出来的实例（对象），实例有一个内部指针proto指向会 Person 构造函数的原型 prototype；换句话说，实例与 Person 构造函数没有直接的关系，而是与构造函数的原型，即 Person.prototype 建立起了连接。通过实例可以访问原型中的属性，但是并不能从实例重写原型的值。

原型规则

  • 所有的引用类型（数组、对象、函数），都具有对象特征，即可自由扩展属性；
  • 所有的引用类型，都有一个proto 属性（隐式原型），属性值是一个普通对象；
  • 所有函数，都具有一个 prototype（显示原型），属性值也是一个普通原型；
  • 所有的引用类型（数组、对象、函数），其隐式原型指向其构造函数的显式原型；（obj.__proto__ === Object.prototype）；
  • 获取对象的某个属性时，若这个对象本身没有这个属性，那么会去它的proto（即它的构造函数的 prototype）中去寻找，直至查找到 Object.prototype 的原型 null；

原型链

每个对象拥有一个原型对象，通过proto指针指向上一个原型 ，并从中继承方法和属性，同时原型对象也可能拥有原型，一层一层递进，最终指向 Object.prototype 的原型对象 null。这种关系被称为原型链 (prototype chain)

person1 从构造函数创建出来，其原型对象指向了 Person.prototype，而 Person构造函数属于对象，其原型指向 Object.prototype，最终 Object.prototype 的原型指向 null。

设计模式

设计模式是一套被反复使用，思想成熟，经过分类和无数实战设计经验的总结。使用设计模式是为了让系统代码可重用，可扩展，可解耦，更容易被人理解且能保证代码可靠性。设计模式使代码开发真正工程化。

六大设计原则

设计模式并不是根据想法随意设计，而是遵循以下规则，才能设计出让人容易理解的代码：

  • 单一职责原则：一个方法只做一件事，一个类只做一个功能；
  • 里氏替换原则：子类可以扩展父类的功能，但不能改变父类原有的功能；
  • 依赖倒置原则：只依赖接口不依赖方法，不关心底层的实现方法；
  • 接口分离原则：把大的接口拆分成小的接口，即功能单一接口；
  • 迪米特原则：函数中传入的参数越少越好，降低耦合度；
  • 开闭原则：面向扩展开放，面向修改关闭，可扩展；

常用设计模式

工厂模式

工厂模式是将其成员对象的实例化推迟到子类中，子类可以重写父类接口方法以便创建时指定独自的对象类型。

抽象的工厂函数，可以将公共的方法写在函数上，而个性化的操作则可以在子类中实现。且外部不许关心内部构造器是怎么生成的，只需调用一个工厂方法生成一个实例即可；

工厂模式适用场景：

  • 对象的构建十分复杂；
  • 需要依赖具体环境创建不同实例；
  • 处理大量具有相同属性的小对象；

单例模式

单例模式也称单体模式，是一个用来划分命名空间并将一批属性和方法组织在一起的对象；提供了一种将代码组织为一个逻辑单元的手段，这个逻辑单元中的代码可以通过单一变量进行访问。

策略模式

策略模式是将定义的一组算法封装起来，使其相互之间可以替换。封装的算法就有一定的独立性，不会随客户端的变化而变化。

中介者模式

中介者模式（Mediator），用一个中介对象来封装一系列的对象交互。中介者使各对象不需要显式地相互引用，从而使其耦合松散，而且可以独立地改变它们之间的交互。

观察者模式

观察者模式也称发布订阅模式，是对象间的一种一对多的关系，让多个观察者对象同时监听某一个主题对象，当一个对象发生改变时，所有依赖于它的对象都将得到通知并自动更新，解决了主体对象与观察者之间功能的耦合。
