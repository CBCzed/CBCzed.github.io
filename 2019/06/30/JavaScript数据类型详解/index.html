JavaScript数据类型

ECMAScript标准种定义了7种数据类型，其又分为6种原始（基本）类型和引用（对象）类型；

原始类型

  • NULL 类型： 只包含一个值：null；
  • undefined 类型： 只包含一个值：undefined；
  • Boolean 类型： 包含两个值true、false；
  • Number 类型： 包含整数、浮点数、Infinity、NaN；
  • String 类型： 一串表示文本值的字符序列；
  • Symbol 类型： 一种实例是唯一且不可改变的数据类型；

引用类型（对象类型）

Object：常用对象Object、数组Array、函数Function、日期Date、正则RegExp；

原始类型与引用类型的区别

1、原始类型的不可变性

原始类型在ECMAScript标准种被定义为 primitive values ，即原始值，代表其值是不可被改变的，原始类型是按值访问的，任何方法都无法修改原始类型的值

如上示例，给String类型的name变量调用字母大写方法，打印name，还是原来的值，调用的方法只是产生新的字符串，给其添加属性，打印结果为undefined，所以不论经过什么操作，原始类型的值都无法被改变

null与undefined的区别

null

null 是特殊的一个对象，表示’无’的对象，即此处没有对象，转为数值时为 0；
null 用法：

  • 作为函数的参数，表示该函数的参数不是对象
  • 作为对象原型链的终点
  • 作为解除引用，便于垃圾回收

undefined

undefined 表示’缺省值’，就是此处应该有一个值，但是还没有定义，转化为数值为 NaN;
undefined用法：

  • 变量被声明了，但没有赋值时，就等于 undefined
  • 调用函数时，应该提供的参数没有提供，该参数等于 undefined
  • 对象没有赋值的属性，该属性的值为 undefined
  • 函数没有返回值时，默认返回 undefined

Symbol类型

Symbol类型为ES6新增的原始类型; 表示独一无二的值。

Symbol数据类型的使用

声明symbol类型变量，需调用 Symbol() 方法，并且可传递一个字符串用于描述，当传递的是一个对象类型的参数，则会调用该对象的toString方法，将其转化为一个字符串，说到底，symbol传递的参数只能是字符串

1、唯一性

当使用 Symbol.for(key) 创建相等的变量时，若已声明的相同的变量，则返回该变量，否则将使用给的的key在全局注册一个新的symbol变量，若想查找是否存在key的symbol变量，可使用 Symbol.keyFor(key) 来查找。

2、原始类型

Symbol 数据类型归属于原始类型。

3、不可枚举性

当使用 Symbol值作为对象的属性名时，可以保证对象不会出现重名属性，但使用 for…in 和 for…of 是无法遍历到 Symbol值的属性，也无法通过 Object.keys()、Object.getOwnPropertyNames() 获取，需使用 Object.getOwnPropertySymbols() 来获取专门的symbol属性，也可以使用 Reflect.ownKeys() 来获取所有属性，包括常规的属性名。

Symbol数据类型的作用

Symbol 的作用非常的专一，换句话说其设计出来就只有一个目的——作为对象属性的唯一标识符，防止对象属性冲突发生。

Symbol应用场景

  • 防止XSS
  • 私有属性
  • 防止属性名冲突

装箱与拆箱

包装类型（临时对象）

存取字符串、布尔值、数字的属性时创建的临时对象称为包装对象；如 new String() new Number() new Boolean() ；
虽说是创建基本类型的临时对象，但跟原始类型还是存在本质的区别：

包装类型与引用类型的区别主要在于对象的生命周期，使用 new 操作符创建出来的引用类型，会在一直存在于内存中，直到执行作用于离开为止，而 new 的包装类型，则在创建完基本类型后，就会马上被销毁。

装箱与拆箱

装箱：基本类型转换成对应的包装类型。
拆箱：引用类型转换为基本类型。

整个过程历经三个步骤，所以当我们对原始类型调用方法时，就会自动进行装箱与拆箱的操作。以上以 String类型进行举例，Number 与 Boolean 也是相同的道理。
引用类型转基本类型，会遵守EMCAScript规定的 toPrimitive (一种算法)原则；一般都是调用 toString 和 valueOf() 方法；当然不同类型的转换规则也不一样：

  • 当引用类型转换为 Number类型，先调用 valueOf，再调用 toString；
  • 引用类型转换为 String类型，先调用 toString，再调用 valueOf；
    若 valueOf 和 toString 都不存在，或者没有返回基本类型，则抛出 TypeError 异常。

JavaScript类型判断方式

typeof

适用场景： typeof 主要适用于除 nul l以外原始类型/及函数的判断，无法用于判断引用类型；

不适用场景：引用类型及 null

instanceof

适用场景：具体的引用类型;

instanceof 检测的是原型，且无法判断所以判断出来的数据也不太准确

toString

toString() 是 Object 的原型方法，调用该方法，默认返回当前对象的 [[Class]] 。这是一个内部属性，其格式为 [object xxx] ，其中 xxx 就是对象的类型。各个类型的转化如下：

类型 转换方法 结果
String Object.prototype.toString.call(‘’) [object String]
Number Object.prototype.toString.call(1) [object Number]
Boolean Object.prototype.toString.call(true) [object Boolean]
Symbol Object.prototype.toString.call(Symbol()) [object Symbol]
Undefined Object.prototype.toString.call(undefined) [object Undefined]
Null Object.prototype.toString.call(null) [object Null]
Function Object.prototype.toString.call(new Function()) [object Function]
Date Object.prototype.toString.call(new Date()) [object Date]
Array Object.prototype.toString.call([]) [object Array]
RegExp Object.prototype.toString.call(new RegExp()) [object RegExp]
Error Object.prototype.toString.call(new Error()) [object Error]
HTMLDocument Object.prototype.toString.call(document) [object HTMLDocument]
window Object.prototype.toString.call(window) [object global]

比较以上三种检测类型的方法， typeof 对原始类型的检测比较准确， instanceof 检测具体的引用类型效果比较明显，而使用 toString 则可以检测任意变量的具体类型。

JavaScript类型转换

因为 JavaScript 是弱类型的语言，所以，类型之间的转换是非常频繁的。
类型转换分为两种，一种是隐式的转换，即程序自动帮我们转换，另一种是强制转换，常常为我们手动进行的类型转换。
强制转换这里就不过多说明，我们来看看隐式转换常常发生的情景：

原始值 转换为Number 转换为String 转换为Boolean

  • 规则二：当一侧为 Number 类型，另一侧为原始类型，则将原始类型转换为 Number 类型;
  • 规则三：当一侧为 Number 类型，另一侧为引用类型，将引用类型和 Number 类型转换成字符串后拼接;
