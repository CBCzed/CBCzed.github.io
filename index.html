JavaScript继承

发表于 2019-09-16 更新于 2020-02-25

导读

什么是继承呢？简单点说就是通过某种方式让一个对象可以访问到另一个对象中的属性和方法。
JavaScript 本质上不能算是一门面向对象的编程语言，因为它对于封装、继承、多态这些面向对象语言的特点并没有在语言层面上提供原生的支持。但是，它引入了原型 (prototype) 的概念，可以让我们以另一种方式模仿类，并通过原型链的方式实现了父类子类之间共享属性的继承以及身份确认机制。

JavaScript内存详解

发表于 2019-07-15 更新于 2020-01-21

导读

你在编写代码时，是否有疑问过，编写的代码存储于哪里呢？
JavaScript 是在创建变量（对象，字符串等）时自动进行了分配内存，并且在不使用它们时"自动"释放。 释放的过程称为垃圾回收；也许是这个"自动"的词语混淆了大家忽略了对内存的管理，然而，了解内存机制是我们进阶 JavaScript 的必然要求；因为很多相关知识的背后都会牵扯到内存管理，比如，浅拷贝与深拷贝的原理、值传递与引用传递、闭包的问题等等；所以了解JS内存机制不仅有助于我们理解 JavaScript 相关的知识，而且还能提高我们的代码质量，打造高性能的应用。
