DOM事件对象

发表于 2020-08-30 阅读次数： 阅读次数： Valine：

导读

　　本文主要是对 JavaScript 的 DOM 对象做一次记录，知识点包括事件的定义、DOM 事件级别、DOM 事件模型与事件流、事件的绑定与移除、事件对象及事件委托

事件

事件本身并不是一个需要定义的技术名词。简而言之，事件就是文档或浏览器通知应用程序发生了什么事情；JavaScript与HTML之间的交互就是通过事件来实现的。
在 HTML 中，元素是自带事件的，不论我们是否给元素绑定函数，都会触发事件。

事件级别

到目前为止，DOM事件有五个级别，分别为：DOM 0级、DOM 1级、DOM 2级、DOM 3级、DOM 4级，但目前DOM 4级还处于草案阶段，还没有成为真正的标准。

DOM 0级

DOM 0级是 JavaScript 指定事件处理程序的传统方式，通过将函数赋值给事件处理程序的属性；其使用方式有两种如下：

  • 行内事件：给标签中的事件属性直接赋值
  • 元素.on事件名=函数

DOM 1级

其实在事件方面是不存在DOM 1级事件的，W3C 组织在制定DOM 1级标准时并没有定义关于事件的相关内容；DOM 1级主要是定义了 HTML 和 XML 文档的底层结构。所以并没有DOM 1级事件的相关定义，但是在面试时，有些面试官就会在这里挖坑。

DOM 2级

DOM 2级新增了两个事件处理程序的方法，分别是用来注册和移除事件的addEventListener和removeEventListener；

DOM 3级

DOM 3级，主要是新增了一些不同的事件类型及自定义事件，新增的事件类型如下：

  • UI 事件，当用户与页面上的元素交互时触发，事件类型有：load、unload、abort、error、select
  • 焦点事件，当元素获得或失去焦点是触发，事件类型有：blur、focus、focusin、focusout
  • 鼠标事件，用户在页面上使用鼠标点击时触发，事件类型有：click、dbclick、mousedown、mouseenter、mouseleave、mousemove、mouseout、mouseover、mouseup
  • 鼠标滚轮事件，当用户使用鼠标滚轮操作时触发，事件类型有：wheel
  • 文本事件，当用户在文本框输入文本时触发，事件类型有：beforeinput、input
  • 键盘事件，用户通过键盘操作时触发，事件类型有：keydown、keyup
  • 合成事件，当 IME(Input Method Editor 输入方法编辑器)输入字符时，触发，更多关于IME，事件类型有：compositionstart、compositionupdate、compositionend
  • 变动事件，当 DOM 结构发生变化时触发，事件类型有：DOMSubtreeModified、DOMNodeInserted、DOMNodeRemoved、DOMNodeInsertedIntoDocument、DOMNodeRemovedFromDocument、DOMAttrModified、DOMCharacterDataModified

以上事件类型中 blur、focus、mouseenter、mouseleave 是无冒泡的，也就是说事件是在捕获时触发

自定义事件需要使用Event或CustomEvent来构造，使用dispatchEvent来触发事件，其构造格式分别如下：

  • Event(‘事件名’)
  • CustomEvent(‘事件名’, {detail: {}, bubbles: Boolean), cancelable: Boolean})，detail表示接收自定义传递的参数，bubbles表示事件是否可以冒泡，cancelable表示事件是否可以取消

DOM事件模型与事件流

DOM事件模型分为捕获与冒泡：

  • 捕获事件，由外向里，即外曾元素先执行，直到目标元素；
  • 冒泡事件，与捕获事件相反，由里向外，即目标元素先执行，再到最外层的元素

DOM事件流，也是DOM事件传播机制，描述的是DOM事件的执行过程，其主要流程为：事件捕获 => 目标元素 => 事件冒泡，具体流程为：window => document => html标签(即document.documentElement) => body => 按层级的标签找到目标 => 再向外执行冒泡

事件绑定与移除

事件的绑定目前主要有两种实现方式，分别是在DOM 0级和DOM 2级事件种定义的：

DOM 0级事件绑定与移除

DOM 0级事件绑定有两种方式：

  • 行内事件，直接在标签的属性上赋值；
  • 元素.on事件名=函数，通过 JS 获取元素后再赋值给事件属性；
    使用行内事件时，是无法移除事件的，这点需要特别注意。

DOM 0级事件的实现原理主要是给元素的事件属性赋值函数，当事件触发时，浏览器会执行绑定的方法。

DOM 0级事件移除的方法比较简单，另元素的事件属性为null即可，即：元素.on事件名=函数

使用DOM 0级事件的绑定方式缺点在于，只能给事件属性赋值一个函数，多次赋值也以最后一次赋值为准。

DOM 2级事件绑定与移除

DOM 2级新增了addEventListener与removeEventListener，分别对应事件的绑定与移除；
addEventListener 语法：
*. 元素.addEventListener(eventName, handler[,useCapture])
*. 元素.addEventListener(eventName, handler[,options])

  1. eventName 事件行为，即要注册的事件名；
  2. handler 事件行为的处理函数；
  3. useCapture 可选参数：true/false; 默认 false，false表示冒泡触发，true 表示捕获触发；
  4. option 可选参数对象，含有以下属性：
    4.1 capture 属性与 useCapture 的功能一样，
    4.2 once: true/false，表示 handler 处理函数是否值触发一次，
    4.2 passive: true/false，表示 handler 处理函数永远不会调用 preventDefault()，即使调用了也会被忽略，

addEventListener的实现原理是将监听事件的处理函数添加到EventTarget上的指定事件类型的事件侦听器列表中，

removeEventListener语法：

  • 元素.removeEventListener(eventName, handler[, useCapture])
  • 元素.removeEventListener(eventName, handler[, options])
  1. eventName 事件行为，即要移除的事件名
  2. handler 事件行为的处理函数，必须使用注册事件时的处理函数
  3. useCapture 可选参数：true/false; 默认 false，false表示冒泡触发，true 表示捕获触发
  4. option 可选参数对象，只包含 capture属性，其功能与useCapture一样

使用addEventListener注册的事件必须通过removeEventListener才能移除事件，且传入的参数必须相同才有移除，若使用的是匿名函数，即使匿名函数的内容是一样的，是无法移除事件的；还有一个参数useCapture/capture这参数配置也必须相同，否则也无法移除事件：

addEventListener与removeEventListener这两个方法是不兼容IE6～8 版本的 IE 浏览器的，需要通过 attachEvent与detachEvent来实现事件的注册与移除，不过由于微软对 IE 的放弃，所以现在基本不用考虑对 IE 的兼容性。

事件对象

定义

事件对象是指，当事件行为触发时，会产生一个当前操作的相关信息对象，并传递给事件处理函数，也就是常见的event对象。
事件对象event是Event类的实例，根据不同的事件类型，会基于Event类创建不同事件对象类，而基于不同的事件对象类，从而产生不同的事件对象。如：鼠标事件对象是MouseEvent类的实例、键盘事件对象是KeyBoardEvent类的实例等等。

不过需要注意一点，事件对象只有在事件处理函数运行期间才会存在，一旦事件处理函数运行结束，事件对象就会被销毁。

常用属性

事件对象的属性有很多，这里只挑选一些常用的做说明，具体的可查阅 MDN 上Event文档的说明

  • type: 触发的事件类型，如 click、change等；
  • target: 事件源，即当前操作的目标元素；
  • currentTarget: 当前绑定事件的目标元素；
  • preventDefault(): 用于阻止元素的默认事件；
  • stopPropagation(): 用于阻止冒泡事件

事件委托

定义

事件委托又称为事件代理，是用一个事件处理程序来管理管理某一类型的所有事件。
事件委托的原理是利用事件的冒泡传播机制，将同一类的事件类型绑定给其父元素，并通过事件对象的事件源(event.target)处理不同的业务。
