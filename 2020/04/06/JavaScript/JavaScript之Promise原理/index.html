什么是Promise/A+

Promise/A+ 规范是一个开放、健全且通用的 JavaScript Promise 标准。其基本要求包括 Promise 的状态及 Then 方法，在说明要求之前，我们先来介绍一下 Promise/A+ 规范相关的术语。

术语

  • Promise：一个拥有 then 方法的对象或函数，其行为需要遵守本规范。
  • thenable：一个定义了 then 方法的对象或函数。
  • 值(value)：值也称为终值，指的是 promise 被解决时传递给解决回调的值，由于 promise 有一次性的特征，因此当这个值被传递时，标志着 promise 等待态的结束，故称之终值。值可以是任何 JavaScript 的合法值（包括 undefined , thenable 和 promise）。
  • 异常(exception)：是使用 throw 语句抛出的一个值。
  • 解决(fulfill)：指一个 promise 成功时进行的一系列操作，如状态的改变、回调的执行。虽然规范中用 fulfill 来表示解决，但在后世的 promise 实现多以 resolve 来指代之。
  • 拒绝(reject)：指一个 promise 失败时进行的一系列操作。
  • 据因(reason)：也就是拒绝原因，指在 promise 被拒绝时传递给拒绝回调的值。

基本要求

Promise状态

Promise 对象有三种状态：

  • Pending 状态表示进行中；可以变为 Fulfilled 或 Rejected 。
  • Fulfilled 状态表示已成功；不可变为其他状态且拥有一个不可变的终值。
  • Rejected 状态表示已失败；不可变为其他状态且拥有一个不可变的据因。

Promise 实例接受一个函数作为参数，我们称该函数参数为 handler ， handler 接受 resolve 和 reject 两个参数，用于改变 Promise 的状态及传入 Promise 的值；

  • resolve 是将 Promise 对象的状态从进行中 Pending 变为已成功 Fulfilled ；
  • reject 是将将 Promise 对象的状态从进行中 Pending 变为已失败 Rejected ；

resolve 与 reject 可以接收任意类型的值作为实参，作为 Promise 对象成功或失败的值。

Promise 对象的状态一旦改变，就不会再发生任何变化，即 Promise 对象的状态由 Pending 变为 Fulfilled 或 Rejected 之后，便不在发生任何的变化。

Then方法

Promise 对象必须提供一个 then 方法以访问其当前值、终值、据因。
then 方法接受两个可选参数： onFulfilled 和 onRejected ；参数类型须为函数，否则会被忽略。

onFulfilled特性：
  • 当 promise 执行结束后其必须被调用，其第一个参数为 promise 的终值；
  • 在 promise 执行结束前其不可被调用；
  • 其调用次数不可超过一次
onRejected特性：
  • 当 promise 被拒绝执行后其必须被调用，其第一个参数为 promise 的据因；
  • 在 promise 被拒绝执行前其不可被调用
  • 其调用次数不可超过一次
多次调用

then 方法可以被同一个 promise实例调用多次：

  • 当 promise 成功执行时，所有 onFulfilled 需按照其注册顺序依次回调；
  • 当 promise 被拒绝执行时，所有的 onRejected 需按照其注册顺序依次回调
返回

then 方法必须返回一个新的 Promise 对象
