JavaScript之this之谜

发表于 2020-03-10 更新于 2020-05-17 阅读次数： 阅读次数： Valine：

导读

　　在 JavaScript 中 this 的指代常常是令人头疼的一个问题，不同于其他语言，如 Java 等面向对象的语言中， this 的含义是明确且具体的，即指代当前的对象，而在 JavaScript 中 this 是动态绑定的或者称运行时绑定的，这就给 this 关键字带来多重的含义，虽然这可能是 JavaScript 作为灵活性语言的标志，但同时也给我们带来了不少困惑，本文将带你揭开 JavaScript 中 this 的迷底，让你不在深陷其谭。

this指向

在查阅网上或者请教别人 this 指向时，总是会告诉我们一句：this 永远指向最后调用它的那个对象，看似简单的一句话，但对 JavaScript 经验尚浅的人来说，还是不太容易理解的，因为 this 在不同的执行环境的表现都会有所不同。
在全局环境中(任何函数体外)，无论是否在严格模式下，JavaScript 的 this 都指向全局对象；

而函数体内部， this 的指向就比较复杂，可以是全局对象、当前对象或者是任意对象，当然也可能会是 undefined， this 最终会指向谁，这完全取决于函数被调用的方式，而 JavaScript 函数的调用方式主要有：

  • 作为普通的函数调用
  • 作为对象的方法调用
  • 作为构造函数调用

ES6箭头函数中的this

ES6箭头函数的 this 指向跟上面所讲的那些情况不太一样，参考《你不知道的 JavaScript (上卷)》一书对箭头函数 this 的解释：箭头函数是根据外层（函数或者全局）作用域来决定 this，我个人对这句话的理解是，箭头函数中的 this 是函数在定义时指向最近一层作用域里的 this。

分析一下这段代码，按照《你不知道的 JavaScript (上卷)》的定义，对象 obj 中的 fn 方法是个箭头函数，所以在定义时就指向了最近一层的作用域，即全局作用域，所以 fn 方法中的 this 指向了全局对象，也就输出了 window； fn1 方法中的定时器的第一个参数采用了箭头函数，该箭头函数中的 this 指向最近一层作用域(此处是局部(函数)作用域)中的 this ，即 fn1 方法中的 this ，而 fn1 方法是一个普通函数，所以其 this 的指向是在函数被调用时决定的，因其是被 obj 调用的，所以 this 指向了 obj 对象，所以定时器箭头函数中的 this 也就指向了 obj ，也就输出了 zed。

ES6 箭头函数是根据当前的词法作用域来决定 this ，具体来说，箭头函数会继承外层函数调用的 this 绑定，无论使用(call、apply、bind等)什么都无法改变 this 的绑定。

this指向的改变

既然 JavaScript 的 this 是动态绑定，那么就可以改变其this的指向，而改变this的指向主要有以下这几种方法：

  • 使用变量保存 this
  • 使用call和apply方法
  • 使用bind方法

bind 方法创建一个新的函数，在 bind 被调用时，这个新函数的 this 被指定为 bind 的第一个参数，而其余参数将作为新函数的参数。
